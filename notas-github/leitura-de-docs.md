
### [About Branches (Sobre Galhos)](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-branches)

> Usado como um hambiente de desenvolvimento outras partes do seu **REPOSITÓRIO**[^], as *branches*, permiten o desenvolvimento de *features*(funções/novas funções), arrumar bugs,
ou simplesmente um hambiente para experimentra novas ideias em um espaço

### [About Commits(Sobre Comitações)](https://docs.github.com/en/pull-requests/committing-changes-to-your-project/creating-and-editing-commits/about-commits)

> Similar a salvar um arquivo editado, o *Commit* salva mudanças em um ou mais arquivos na sua *branch*. Fazendo o *Git*(distributed version control system), que é totalmente diferente de *GitLab, GitHub, etc*, registrar um ID(IDentification) único para cada *commit*, chamada **SHA** ou **hash**, que identifica:

* *`The specific changes`*
* *`When the changes were(forão) made`*
* *`Who create teh changes`*
